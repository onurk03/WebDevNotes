Images
  1. Use GIF for large areas of single color, limited color space, can do transparency with jagged edges.
  2. SVG (scalable vector graphic) is good for complex drawings, logos, icons, etc. as it can be scaled without losing quality. (small file size)
  3. JPG is good, but it has to be resized and compressed. Manually or software.
  4. PNG is good at compression and images that need transparency, depending on use.

    5. Check image files to make sure they're downloaded fast on website

Make copies of the image in different resolutions for different screen sizes, and use that as options in the HTML:

  1. This is done by using srcset="" and listing the different resolution copies of the image followed by a space and then the resolution (1x, 2x or 480w, 1980w for width)
  2. The "sizes" attribute can be used to list which size image to use at which media query.
  3. If we want to use a different image for different screen sizes to avoid cropping, we wrap the img element inside a "picture" element. Then we use the "source" element and "srcset" attribute inside it to list the alternatives. A "media" attribute can be used to define the min-width of a viewport in which the alternative image will be used
  4. A "figcaption" element can be used to caption images, then both the img and figcaption elements can be wrapped inside a "figure" element to give the browser and search engines info about the content.
Audio
  1. The <audio> element can be used to place audio on a web page.
  2. The "controls" attribute can be added to allow browser default media controls.
  3. The "loop" attribute can be used to make the media loop after it's played.
  4. The "autoplay" attribute can be used to make the media play when the page is opened.
Videos
  • Make sure videos are compressed with the right codec. (H.264 is good but patented)
  • AV1 or WebM are good options. Adaptive bitrate streaming can be used to switch resolutions depending on internet speed and display.
  • Make sure to include captions and subtitles in the webvtt format using the <track> element.
  • Types of "kind" attributes for videos: Captions, subtitles, descriptions, chapters.
  • Embedding other media services can be better for adaptive bitrate streaming rather than using the video element.

Identifying Content
  • Lang, dir, and charset attributes are used to specify content language inside the html element or others.
  • lang="en-US" inside the html or any other element, <meta charset="UTF-8"> inside the head element
  • <div> or <span> elements can be used to wrap a group of elements. It's not a good idea to use them for everything.
HTML Structures
  • main

    - Wraps around the main content of the page. (used once)
  • header/footer

    - Used to wrap header or footer.
  • article

    - Used to wrap articles, can also be used for teaser cards, or social media posts.
  • section

    - Used to mark the sections of long content. (Essays with subheaders)
  • aside

    - Marks content that is off to the side or not the main attraction.

Structuring Tabular Data
  • <table> - Table (Wraps the whole table)
  • <tr> - Table row (Wraps a set of elements in the same row)

    - Attributes: colspan, rowspan, headers
  • <th> - Table Header (Defines a header for the column)

    - Attributes: colspan, rowspan, scope
  • <td> - Table Data (Marks the actual bits of data)
